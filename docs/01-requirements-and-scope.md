# 要件・スコープ

## 目的

本システムは、メールボックスに届いた特定条件のメールを .NET 8 コンソールアプリで取得し、メール内容を連携用 API へ送信する処理を検証するためのサンプルである。

検証では、メール投入、メール取得、API 連携、結果確認までをローカル環境で再現できることを重視する。

## 対象機能

| 区分 | 内容 |
| --- | --- |
| メールサーバ | Docker Compose でテスト用 SMTP / IMAP サーバを起動する。 |
| テストメール投入 | .NET 8 コンソールアプリから SMTP でテストメールを送信する。 |
| メール取得 | .NET 8 コンソールアプリから IMAP でメールボックスへ接続し、対象メールを取得する。 |
| 抽出項目 | Message-Id、送信者、件名、本文、受信日時を抽出し、必要に応じてメールの一部分だけを API 送信用データへ加工する。 |
| キュー制御 | API のリクエスト型に合わせた送信用データを内部キューへ追加し、追加完了かつキュー残データなしになるまで処理する。 |
| API 連携 | 内部キューから取得した送信用データを HTTP API へ POST する。 |
| 証跡保存 | API は受信リクエストを SQLite に保存し、GET API で確認できるようにする。 |
| バッチログ | バッチは処理開始、メール取得件数、API 連携結果、エラー内容を `.log` ファイルへ出力する。 |
| 運用制御 | 二重起動を防止し、処理台帳・実行履歴を保存して、実行結果と異常メトリクスをメール通知する。 |

## 初期スコープ外

- 本格的な CRUD 画面
- 認証・認可
- 添付ファイル処理
- 本番運用を前提にした高可用化、監視、アラート
- メール本文の高度な HTML パースや業務ドメイン固有の解析

## 技術方針

| 項目 | 推奨 |
| --- | --- |
| コンソールアプリ | .NET 8 Console |
| API | ASP.NET Core Web API |
| メール取得 | IMAP |
| メール送信テスト | SMTP |
| DB | SQLite から開始 |
| ログ | Serilog |
| Docker | docker-compose |
| API 確認 | DB 保存 + GET API |

## 成功条件

1. Docker Compose でメールサーバと API が起動できる。
2. テストメール送信アプリで任意のテストメールをメールボックスへ投入できる。
3. バッチアプリが IMAP で対象メールを取得できる。
4. バッチアプリが抽出・加工したデータを API リクエスト型として内部キューへ追加できる。
5. バッチアプリがキューに残データがある限り API へ POST し続け、追加完了かつ残データなしで終了できる。
6. API が受信内容を SQLite に保存できる。
7. GET API または SQLite DB から保存済みデータを確認できる。
8. バッチ処理の開始、件数、キュー追加結果、API 結果、エラーが日次ログファイルに残る。
9. IMAP / API の一時障害を設定回数まで指数バックオフで再試行し、処理結果に応じた終了コードと通知を返せる。
